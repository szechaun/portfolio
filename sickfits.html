Sick Fits

The Project

Sick Fits is a full-stack online clothing store that allows users to search, sell, add to cart, and checkout their favorite items. Built using React, Next.js, and styled components, the application integrates Apollo Client for GraphQL queries and mutations, and Keystone.js to manage data via an admin interface. The project consists of six main relational models: Users, Items, Orders, CartItems, OrderItems, and Roles, which are tightly connected through GraphQL.

Tools

React, Next.js, Styled Components, Apollo Client, Keystone.js

Skills

Full-Stack Development, GraphQL, UI/UX Design

The Process

Back-End Development

I set up the GraphQL schema and Keystone.js admin interface to handle relational data, defining relationships between users, products, and orders.

Front-End Development

With the back-end structured, I moved on to building the front-end with React and Next.js. I chose Next.js for its server-side rendering capabilities to improve SEO and performance, ensuring a smooth and responsive user experience.

The Pitfalls

Challenges & Solutions

One of the biggest challenges was optimizing GraphQL queries for the checkout process, as handling relational models efficiently led to performance bottlenecks. I optimized the Apollo Client caching mechanism using `refetchQueries` to ensure data consistency.

Iterations

The first version of Sick Fits focused on core functionality, such as user authentication and adding items to the cart. Later iterations involved refining the user interface and adding features like search filters and checkout options. During these iterations, I continuously tested the GraphQL schema to ensure it handled relational data properly and optimized API calls to improve performance, especially as more users interacted with the platform simultaneously.

Reflection

Building Sick Fits taught me a lot about managing relational data with GraphQL and working with modern web technologies like React and Next.js. If I were to improve the project, I would focus on better optimization for mobile users and explore using GraphQL subscriptions to handle real-time data updates more efficiently. Overall, the project has been a great learning experience in full-stack development.
